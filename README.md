# Exercice Pratique - Développeur React

## Contexte

Vous intégrez une équipe qui travaille sur une application de gestion de tâches. Le code existant fonctionne, mais l'équipe rencontre des problèmes de performance et de maintenabilité.

## Objectif

Analyser le code fourni et proposer des améliorations concrètes pour :
- Optimiser les performances et les re-rendus
- Améliorer la testabilité
- Augmenter la qualité de vie des développeurs (DX)

## Instructions

1. **Analyse** (30-45 min)
   - Examinez le code dans `src/components/TaskManager.tsx` et `src/components/TaskList.tsx`
   - Identifiez les problèmes de performance, testabilité et maintenabilité
   - Documentez vos observations dans `ANALYSIS.md`

2. **Refactoring** (1h-1h30)
   - Proposez et implémentez des améliorations
   - Vous pouvez créer de nouveaux fichiers si nécessaire
   - Documentez vos choix dans `IMPROVEMENTS.md`

3. **Tests** (30 min)
   - Ajoutez des tests pour démontrer la testabilité de votre solution
   - Utilisez le framework de test de votre choix

## Ce que nous évaluons

- Compréhension des Pure Components et optimisation des rendus
- Priorisation des améliorations / correctifs et argumentation de celle-ci
- Capacité à écrire du code testable
- Qualité de la séparation des responsabilités
- Connaissance des patterns React modernes
- Clarté de la documentation et de la communication
- Si usage de Generative AI, approche et niveau d'utilisation de celle-ci

## Livrables attendus

1. `ANALYSIS.md` - Vos observations sur le code existant
2. Code refactorisé avec vos améliorations
3. `IMPROVEMENTS.md` - Explication de vos choix techniques
4. Tests unitaires

## Setup

```bash
npm install
npm start
npm test
```
